## 1. プロダクト概要

Neural DSP Archetype シリーズと同等のコンセプトを持つ、ギター用アンプシミュレーター / マルチエフェクトソフトウェア。
ニューラルネットワークによるアンプキャプチャ(NAM 互換)をコアエンジンとし、ペダルボード、キャビネット IR、グラフィック EQ、テープエコー、リバーブ、チューナー、メトロノームを統合する。

- 用途: **商用販売**(まず個人利用の範囲で完成・検証してから商用化する。implementation.md 参照)
- ビジュアル: 初期フェーズは仮 UI(機能優先)。本番用 3D レンダ UI は後続フェーズで差し替え可能な設計とする(design.md §10)。

## 3. NAM 技術の基礎

本製品のコア技術である NAM(Neural Amp Modeler)の背景情報。設計判断(§8.1)およびライセンス方針(§4)の根拠となる。

### 3.1 NAM とは

- 実機のギターアンプ / ペダル / アウトボード等の**入出力特性をニューラルネットに学習させ、音と弾き心地をリアルタイム再現する**オープンソース技術
- 学習には「アンプへ送る元の DI / テスト信号」と「実機を通して録音した音」の 2 つを使い、その対応関係(歪み方、圧縮、倍音付加、音量変化への反応)を学習する
- 回路を部品単位で計算する従来型アンプシミュレーターではなく、**特定の実機・セッティングを音からキャプチャするデータ駆動方式**
- 学習結果は `.nam` ファイル(ネットワーク構造・設定・重みを格納した公開仕様のフォーマット。WaveNet / LSTM 等)として書き出される

### 3.2 構成要素とライセンス

| 構成要素 | 内容 | ライセンス |
|---|---|---|
| トレーナー (neural-amp-modeler) | モデル学習(Colab / ローカル Python) | MIT |
| Core DSP (NeuralAmpModelerCore) | リアルタイム再生エンジン。**本製品が組み込むのはこれ** | MIT |
| 公式プラグイン | 現行の公式配布版は「**Gateway**」という名称(VST3 / AU / スタンドアロン)。**iPlug2 製であり、リポジトリ NeuralAmpModelerPlugin は本製品のフォーク元** | MIT |

- MIT License のため、コードの使用・改変・配布・販売が可能。ただし**コードまたはその実質的部分を配布する場合、著作権表示と MIT ライセンス文の同梱が必要**(§4.1、implementation.md Phase 8 チェックリスト対応)
- 公式も NAM 技術の製品組み込み用途を案内しており、**NAM を組み込んだ商用ソフト / ハードの開発は正当な利用形態**

### 3.4 スナップショット型とパラメトリック型

- 流通している通常の `.nam` は「**ある特定のノブ設定をキャプチャしたスナップショット**」。モデル上で Gain や EQ を回しても、実機ノブ全域の挙動を再現するわけではない
- 公式にも通常プラグインは「snapshot plugin」と位置づけられ、ノブ全域を条件として学習する**パラメトリック・モデリング**は別の仕組みとして存在する
- → 本製品の設計判断(§8.1「入力トリム + デジタルトーンスタック」、複数ゲイン段キャプチャのモーフィング構想 §18)はこの制約に対応するもの

### 3.5 実用上の注意 — 入力レベル依存性

- NAM モデルは**入力レベルに対する歪み方まで学習している**ため、キャプチャ時とユーザー環境で入力ゲインが大きく異なると、想定より歪みすぎたり、クリーンになりすぎたりする(アナログ電圧とデジタル音量の対応が環境ごとに異なるため)
- → 本製品への反映: 入力メーターと推奨レベルガイドの表示(§8.1)。自前キャプチャ時(§16)は**録音レベルをメタデータに記録**し、全同梱モデルでレベル基準を統一する

### 3.6 類似技術との比較(参考)

NAM / IK Multimedia TONEX / Neural DSP Neural Capture(Quad Cortex / Nano Cortex)は、いずれも「実機の応答をニューラルネットで学習するデータ駆動型キャプチャ」という同ジャンルの技術。

| 項目 | NAM | TONEX | Neural Capture |
|---|---|---|---|
| 公開性 | オープンソース(構造・重み・仕様を確認可能) | IK 独自(AI Machine Modeling) | Neural DSP 独自 |
| 学習場所 | PC / Colab 等 | TONEX Modeler (Mac/PC) | V1: Cortex 本体 / V2: Cortex Cloud |
| モデル形式 | `.nam`(公開仕様) | Tone Model | Neural Capture |
| 相互互換 | **3 方式間にネイティブ互換なし**(同一実機を各方式で個別にキャプチャすることは可能) | | |

- 各方式はネットワーク構造・学習方法が異なる別実装であり、「同じ技術」ではない
- **本製品が NAM を採用する理由**: オープンな公開仕様(`.nam` をユーザーが自由にロードできるエコシステム)、MIT ライセンスによる商用組み込みの明確な許諾、学習パイプラインを自前で運用できること

### 3.7 アーキテクチャ世代: A1 と A2

A1 / A2 は「NAM モデルを構成するニューラルネットの標準レシピの世代」であり、`.nam` ファイルフォーマットのバージョンとは別の概念。**A2 は 2026 年 6 月 2 日に正式リリースされ、新しい標準になった**(A1 も廃止されず継続サポート)。

| 項目 | A1 | A2 |
|---|---|---|
| 位置づけ | 従来の標準 | 2026-06 以降の新標準 |
| 従来の名称 | Standard / Lite / Feather / Nano(→ A1-standard 等に改称) | A2 |
| 主な活性化関数 | Tanh | LeakyReLU(軽量) |
| モデルサイズ | 1 ファイル = 1 固定サイズ | **SlimmableContainer に複数サイズ(標準は small / large)を格納**し、再生側が CPU 余力に応じて選択 |
| 設計思想 | 実績のある従来設計 | CPU 効率と精度のトレードオフを改善(「必ず高音質」ではない) |

**互換性(本製品の実装方針の根拠)**:

| 再生環境 | A1 モデル | A2 モデル |
|---|---|---|
| 現行の A2 対応 Core(v0.5.2+、推奨 v0.5.4+) | ○ | ○ |
| A2 対応前の古い Core | ○ | 原則 × |

- 新しい Core は古い A1 モデルを読めるが、古い Core は A2 を読めない → **本製品は最新 Core を採用し A1+A2 両対応とする**(A2 のみ対応にすると既存 A1 資産が使えず実用性が大きく下がる)
- **A1 → A2 の重み変換は不可**。A2 化にはキャプチャ音源(DI + 実機出力)からの**再学習**が必要 → 自前キャプチャの録音 WAV(§16)は必ず保管し、将来の再学習に備える
- 実装上の注意:
  - モデル判定を `"A1"` / `"A2"` の文字列決め打ちにしない(A1 の architecture は `WaveNet`、A2 は `SlimmableContainer` 内に WaveNet)。**公式 Core のローダーと能力照会 API に委ねる**
  - A2 のサイズ数を 2 個と決め打ちしない(Core v0.5.4 のサイズ・ブレークポイント取得機能で動的取得)
  - 回帰テスト対象: A1-standard / lite / feather / nano、A2-small / large、ブロックサイズ・サンプルレート変更、モデル切替、不正ファイル。CPU 計測には Core 付属の `benchmodel` を利用

### 4.3 同梱素材の調達方針(基本方針)

1. **アンプモデル: 自前キャプチャを基本とする**(手順は §16)。実機はレンタル / スタジオ借用で調達。自分でキャプチャしたモデルは自らの権利物となる
2. **キャビ IR: 自前収録を基本とする**。マイク位置 × 距離のグリッド収録が、§9 の「IR マトリクスブレンド」用データ作成をそのまま兼ねる
3. 開発・検証中は TONE3000 等の公開モデルを個人利用の範囲で使用可(**製品ビルドには含めない**)
4. 補完策: 商用可を明記するモデル作者や無料 IR 制作者への直接許諾交渉

### 4.4 商標

- 実在アンプ名 (Fender / Marshall 等) を製品名・UI・マーケティングに使用しない。「〜風」の独自名称にする(音色自体には著作権が及ばないというのが一般的な理解だが、商標・不正競争の観点から名称使用は避ける)
- Neural DSP / Archetype 等の名称・ロゴ・固有アートを模倣しない。操作体系(上部グローバルバー + 中央機材画面 + 下部ナビ)の参考は可だが、色・形状・イラスト・アイコンはオリジナルとする

## 5. シグナルチェーン(全体構成)

```
[Input Gain] → [Noise Gate] → [Transpose(Pitch Shift)]
 → [ペダルボード: COMP → OD1 → OD2 → MOD]
 → [アンプ (NAM エンジン + トーンスタック)] ×3 切替
 → [キャビネット / IR (2マイク + 外部IR)]
 → [9バンド グラフィック EQ (+HPF/LPF)]
 → [ECHO(テープエコー)] → [REVERB]
 → [Doubler] → [Output Gain]
```

- 各ブロックは個別バイパス可能(クリックノイズ防止のクロスフェード付き)。
- Input Mode: **初期リリースは Mono のみ**(Stereo 入力対応は将来バージョン。§18-11 で決定)。
- チューナーはチェーン先頭(Input Gain 後)からの信号を解析。チューナー起動時に出力ミュート可。

## 7. 機能仕様 — ペダルボード

各ペダルはフットスイッチで ON/OFF、LED 点灯表示。

| ペダル | パラメータ | 実装方式 |
|---|---|---|
| COMP | Comp / EQ / Volume / Mix | フィードフォワード型コンプ + ティルト EQ + パラレルミックス |
| OD1 (TS系) | Volume / Gain / Tone / Mid Boost | ソフトクリッピング + プリ/ポストフィルタ(2〜4 倍オーバーサンプリング) |
| OD2 (別キャラ) | Volume / Gain / Cut | ハードクリップ寄りの歪み + ハイカット |
| MOD | Speed / Intensity / モード切替 (Tremolo / Vibrato / Chorus) | LFO + 可変ディレイライン / 振幅変調 |

## 8. 機能仕様 — アンプセクション

- **3 アンプ切替: Clean / Crunch / Lead**。切替はクロスフェード。
- コアは NAM モデルのリアルタイム推論。**A1 / A2 両アーキテクチャに対応**(§3.7)。
- **モデル品質切替**: A2 モデルはファイル内に複数サイズ(標準では small / large)を持つため、品質(CPU 負荷)切替 UI を設ける。サイズ候補はモデルから動的に取得し(2 個と決め打ちしない)、固定サイズの A1 モデルロード時は切替 UI を無効化する。

### 8.1 NAM 方式の設計判断

NAM キャプチャはスナップショット(§3.4)のため、ノブを効かせるハイブリッド構成とする:

```
[入力トリム(=Gainノブ)] → [NAMモデル推論]
 → [デジタルトーンスタック: Bass / Middle / Treble / Presence] → [Master]
```

- Gain: モデル前段の入力レベルで歪み量を制御(実機の挙動と厳密には異なることを許容)
- 将来オプション: ゲイン段違い複数キャプチャのモーフィング(§18-3)
- トーンスタック: 実機回路系(Fender/Marshall 型)伝達関数を IIR で実装
- ⚠️ NAM モデルは**入力レベルに対する歪み方まで学習している**(§3.5)ため、入力レベルキャリブレーション機能(Input ノブ + 入力メーターでの推奨レベル表示、将来的には dBu 基準のキャリブレーション設定)を設ける(UI 配置は §18-12)

### 8.2 アンプごとのコントロール

| アンプ | コントロール |
|---|---|
| Clean | Volume / Bass / Cut / Master / Level |
| Crunch | Bright / Gain / Bass / Middle / Treble / Shift / Master / Presence / Output |
| Lead | Gain / Bass / Middle / Treble / Presence / Master / Output(モデル選定後に確定) |

- **アンプ⇔キャビのリンクウィジェット(確定仕様)**(画面下部中央): アンプヘッドとキャビネットは Clean / Crunch / Lead の 3 セット。**リンク ON でアンプ切替に対応キャビが連動**し、**リンク解除でアンプとキャビを自由に組み合わせ**られる(例: Clean アンプ × Lead キャビ)。UI 詳細は design.md §6。

## 9. 機能仕様 — キャビネット / IR セクション

- **キャビネットは 3 種**(Clean / Crunch / Lead の各アンプに対応)。リンク ON でアンプと連動、リンク解除で自由組み合わせ(§8.2)。
- **2 マイク構成**。各マイク独立で:
  - マイク種類: Dynamic 57 / Ribbon 121 / Condenser 87(最低 3 種、独自名称)
  - Position: 0.0〜1.0(キャップ ⇔ エッジ)/ Distance: 0.0〜1.0
  - Mic Level: -60〜+12 dB / Pan: L〜C〜R / Phase 反転 / ON/OFF / ←→ マイクプリセットブラウズ
- 実装: マイク × ポジション × 距離グリッドで収録した **IR マトリクスの補間ブレンド** + WDL ConvolutionEngine(AudioDSPTools の ImpulseResponse 経由、ゼロレイテンシ)
- 外部 IR ロード: WAV (44.1k〜96kHz, mono/stereo)、自動リサンプリング、長さ選択(最大 8192 サンプル)
- キャビセクション全体バイパス(実キャビへ送る用途)

## 10. 機能仕様 — EQ / ECHO / REVERB / チューナー / メトロノーム

### 10.1 グラフィック EQ

| 項目 | 仕様 |
|---|---|
| バンド | 9 バンド: **65 / 125 / 250 / 500 Hz / 1k / 2k / 4k / 8k / 16k Hz**、各 ±12 dB フェーダー。フェーダー上に現在値 (0.0 dB) を表示 |
| HPF | 周波数ノブ(OFF〜約 1 kHz)+ ON/OFF トグル。OFF 時は「HPF : OFF」表示 |

### 10.2 ECHO — テープエコー

多連ヘッドのテープエコーをモデリングする。

## 16. 同梱素材の制作手順(アンプキャプチャ / IR 収録)

自前キャプチャ(§4.3 方針)の具体的な手順。1 アンプ 1 セッティングあたり、収録 15 分 + 学習 15〜60 分が目安。実施タイミングは implementation.md Phase 7。

### 16.1 キャプチャの基本原理

NAM のキャプチャは「**標準テスト信号をアンプに通し、入力(既知のテスト信号)と出力(録音)のペアからニューラルネットを学習させる**」方式。

```
[テスト信号 WAV] → 再生 → [アンプ] → 録音 → [出力 WAV]
                     ↓
        入力WAV + 出力WAV → NAMトレーナーで学習 → .nam ファイル
```

本製品のアーキテクチャ(アンプ NAM + 別途 IR キャビ)に合わせ、**キャビ / マイクを含めない「DI キャプチャ」**(ロードボックス経由)を標準とする。

### 16.2 必要機材

| 機材 | 役割 | 例(価格目安) |
|---|---|---|
| キャプチャ対象アンプ | 実機(レンタル可) | — |
| **ロードボックス(リアクティブ型必須)** | 真空管アンプのスピーカー出力を安全に受け、ライン信号化 | Two Notes Torpedo Captor X(約6万円)、Suhr Reactive Load(約7万円)、Fryette PS-2A |
| **リアンプボックス** | インターフェースのライン出力を楽器レベル / ハイインピーダンスに変換してアンプ入力へ | Radial ProRMP(約2万円)、Little Labs Redeye |
| オーディオインターフェース | テスト信号再生 + アンプ出力録音 | MOTU M2、Focusrite Scarlett 2i2(約2〜3万円)。より高精度なら RME Babyface Pro |
| ケーブル類 | TS / TRS / XLR | — |
| (任意)学習用 GPU | ローカル学習の高速化 | NVIDIA GPU 搭載 PC、または Apple Silicon Mac(MPS 対応)。無ければ Google Colab の無料 GPU で可 |

⚠️ **安全上の注意: 真空管アンプのスピーカー出力を無負荷(ロードボックスもキャビも無し)で動作させると出力トランスが焼損する。ロードボックス接続を必ず先に確認してから電源を入れること。**

### 16.3 結線

```
インターフェース OUT 1 → リアンプボックス → アンプ INPUT
アンプ SPEAKER OUT → ロードボックス(→ 内蔵キャビシムは必ず OFF)
ロードボックス LINE OUT → インターフェース IN 1
```

- キャビシミュレーション / IR 機能付きロードボックス(Captor X 等)は**必ずキャビシム OFF の DRY 出力**を使う(キャビは本製品の IR セクションが担当するため)
- モニターは録音ソフトのソフトウェアモニタリングで行う(ラウドネスに注意)

### 16.4 テスト信号の再生と録音

1. NAM 公式の標準入力信号 **`v3_0_0.wav`**(約 3 分 10 秒。先頭にディレイ校正用バースト、ESR 検証用区間、ノイズ / スイープ / ギター実演を含む)を入手する
   - 配布元: NeuralAmpModeler の公式ドキュメント / トレーナーのリンク(GitHub: `sdatkinson/neural-amp-modeler`)
2. DAW(Reaper 等)で `v3_0_0.wav` をトラックに配置し、OUT 1 から再生
3. アンプを目的のセッティング(例: Crunch チャンネル、Gain 5)に合わせる
4. **録音レベル調整**: 最大音量部でもインターフェース入力がクリップしないこと(ピーク -6〜-3 dBFS 目安)。**ゲート / EQ / リミッター等の処理は一切かけない**
5. 録音: **48 kHz / 24 bit / モノラル WAV**、テスト信号と完全に同じ長さ(先頭・末尾を信号と同一タイミングでトリム。トレーナーが先頭バーストで遅延を自動校正するため、多少のズレは許容される)

### 16.5 学習(トレーナーの選択肢)

| 方法 | 手順 | 所要時間 | 向いている人 |
|---|---|---|---|
| **① GUI トレーナー(ローカル)** | `pip install neural-amp-modeler` → ターミナルで `nam` を実行 → GUI で入力 WAV / 出力 WAV を指定 → Train | NVIDIA GPU: 10〜20 分 / Apple Silicon: 20〜60 分 / CPU: 数時間 | 本製品開発では**これを標準**とする(枚数をこなすため) |
| ② Google Colab(公式ノートブック) | ブラウザで公式 Colab を開き、WAV 2 本をアップロード → 全セル実行 | 無料 GPU で 15 分前後 | GPU が無い場合 |
| ③ CLI / Python API | `nam-full` 等で設定 JSON を指定してバッチ学習 | — | 複数セッティングの一括処理(16.6) |

- **アーキテクチャ選択**: トレーナー(neural-amp-modeler v0.13.0 以降)の既定は **A2**。新規キャプチャは A2 を標準とし、旧環境向け互換が必要な場合のみ A1(旧 Standard / Lite / Feather / Nano = A1-standard 等)を選択する(§3.7)
- **製品同梱**: A2 を基本とする。A2 は small / large の複数サイズを 1 ファイルに持つため、CPU 負荷検証(§14)はサイズ切替と合わせて行う
- エポック数: 既定(100)で十分なことが多い
- **品質指標 ESR**(Error-to-Signal Ratio): 学習完了時に表示。**0.01 未満 = 優秀、0.05 未満 = 実用**。0.1 を超える場合は録音レベル・クリップ・トリムずれを疑い再収録
- 出力: `.nam` ファイル(重みを含む JSON)。本製品のエンジン(NeuralAmpModelerCore)でそのままロード可能

### 16.6 本製品向けの収録プラン

| 目的 | 収録内容 |
|---|---|
| 3 アンプ(Clean / Crunch / Lead) | 各アンプの代表セッティングを 1 キャプチャずつ(トーン系ノブは 12 時、Gain は各チャンネルの美味しい位置) |
| Gain ノブ高度化(将来、§18-3) | 同一アンプで **Gain 位置を 3〜5 段階変えて複数キャプチャ**(例: 9時 / 12時 / 3時)し、モーフィング用データとする |
| 検証用 | ESR 検証区間の再生比較 + 実ギターでの A/B 試奏を各キャプチャで実施 |

- キャプチャ時は**アンプの実セッティング(全ノブ位置)を写真記録**し、モデルファイル名とメタデータ(JSON)に残す(例: `crunch_gain12h_bass12h_a2.nam`)
- トーンスタック(§8.1)はキャプチャ後段のデジタル実装なので、**キャプチャ時のアンプ側 EQ は基準位置(12 時)固定**を推奨。これによりデジタルトーンスタックの効きが素直になる
- **キャプチャ録音 WAV(入力 / 出力ペア)は必ず保管する**(将来のアーキテクチャ更新時の再学習に必要。§3.7)

### 16.7 キャビ IR の自前収録(概要)

アンプキャプチャと同じ機材構成で、ロードボックスの **THRU(スピーカー出力)→ 実キャビ → マイク → インターフェース**に差し替え、サインスイープを再生してデコンボリューションで IR を得る(Voxengo Deconvolver、REW、Reaper の ReaVerb 等)。

- §9 の Position / Distance ブレンド用に、**マイク 3 種 × 位置 5 点 × 距離 3 点程度のグリッド収録**を行う
- 詳細手順は実施前に別ドキュメント化する(残タスク)
