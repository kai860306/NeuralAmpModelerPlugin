## 1. デザイン原則

1. **1 画面 = 1 機材のメタファー**: Neural DSP 同様、各タブに実機(ペダル群 / アンプヘッド / キャビ / ラック)が 1 台鎮座するレイアウト。抽象的なスライダー羅列にしない。
2. **常時見える共通枠**: グローバルバー(上)と下部バーはどのタブでも固定。演奏中に頻繁に触る Input / Gate / Output / チューナー / タップテンポへのアクセスを 1 クリックに保つ。
3. **仮 UI でも情報設計は本番と同一**: フェーズ 2 は「見た目の差し替え」だけで済むよう、配置・サイズ比・操作は本設計で確定させる。
4. **マウス最優先、キーボードは補助**: DAW ユーザーの操作習慣(縦ドラッグ、ダブルクリックでリセット)に従う。

## 2. ウィンドウ / 座標系

| 項目 | 値 |
|---|---|
| 基準解像度 | 1200 × 950 px(参考画像 1 の縦横比に準拠) |
| リサイズ | 比率固定スケーリング 50%〜200%。IGraphics の描画スケール変更で実装(固定アスペクト) |
| 高 DPI | 論理座標で設計し IGraphics(Skia バックエンド)のスケーリングに委譲。画像アセットは @2x を用意 |
| 最小構成 | 1200×950 を 100% とし、フォント・ノブは論理 px 指定 |

## 3. A. タブナビゲーション

- 5 アイコン(左から): ①ペダルボード ②アンプ ③キャビ/IR ④EQ ⑤ディレイ/リバーブ
- 状態: 選択中 = 背景ハイライト + 下部ドット / 非選択 = アウトラインのみ
- 切替: クリック、または Cmd/Ctrl + 1〜5 のショートカット
- アイコンは SVG(仮 UI)→ 本番でも流用可能なピクトグラム(波形 / アンプ / スピーカー / フェーダー / 砂時計)

## 4. B. グローバルバー

| 要素 | UI | 挙動 |
|---|---|---|
| 入力メーター | 縦バー(バー幅 6px) | ピークホールド 1.5s、クリップで赤 |
| INPUT / OUTPUT | 小ノブ (56px) + 下に dB 値 | §7 のノブ共通挙動 |
| GATE | トグル + 小ノブ + dB 値 | OFF 時はノブをディム(操作不可) |
| TRANSPOSE | 小ノブ + 「0 st」表示 | 1 半音刻みでスナップ。0 以外のとき値をアクセント色表示(レイテンシ増の注意喚起) |
| INPUT MODE | Stereo/Mono 縦トグル | **初期リリースは Mono 固定**(Stereo 選択肢はディム表示、将来対応: spec §18-11) |
| プリセットブラウザ | ←→ ボタン + ドロップダウン(幅 420px) | 未保存変更で名前の先頭に `*`。ドロップダウンはフォルダ階層表示 |
| DELETE / SAVE / SAVE AS / ⋮ | テキストボタン | ファクトリープリセット選択中は DELETE / SAVE をディム(参考画像 1 と同挙動) |
| DOUBLER | トグル + 小ノブ + ms 表示 | OFF 時ノブとラベルをディム(参考画像の 6.00 ms のグレー表示) |

## 5. C. メインコンテンツ(タブ別)

### 5.1 ペダルボード

- 4 ペダルを等間隔横並び(1 ペダル約 270×520px)。
- ペダル構成: 上部にノブ 2×2(MOD は 2 + モード切替スイッチ)、中央に機種名、下部にフットスイッチ + LED。
- フットスイッチ: クリックで ON/OFF。LED は ON = シアン点灯。
- ペダル左右の「> >>>> >」矢印(参考画像 4)は将来のペダル順序入替 / ルーティング表示用の飾りとして予約(初期リリースでは非機能)。

### 5.2 アンプ

- 選択中アンプのヘッドを 1 台大きく表示(幅約 1100px)。
- アンプ切替: 下部リンクウィジェット(§6)のアンプアイコン 3 つ、または左右スワイプ矢印。切替時 150ms クロスフェード(音声・画像とも)。
- コントロールパネル帯にノブを横一列配置。ノブ間隔は実機比率を尊重。
- POWER トグル(アンプセクションバイパス)+ パイロットランプ。

### 5.3 キャビネット / IR

- 中央: アンプヘッド + キャビの 3D ビュー(仮 UI ではキャビ正面図)。
- 左右対称に 2 マイクのコントロール群(参考画像 1 準拠):
  - 上段: ←→(マイクプリセット送り)、マイク選択ドロップダウン、電源ボタン
  - 中段: POSITION / DISTANCE ノブ(値 0.000〜1.000 表示)
  - 下段: MIC LEVEL / PAN ノブ、PHASE ボタン(Ø アイコン、反転中はハイライト)
- 外部 IR: マイク選択ドロップダウン内の「Load IR...」項目 + ドラッグ&ドロップ受付(キャビ画像上に WAV をドロップ)。
- キャビ全体バイパスボタン。

### 5.4 EQ

- ラック 1U 機材風の横長パネル(参考画像 2 準拠)。
- 左: HPF ノブ + ON/OFF トグルスイッチ、「HPF : OFF / 数値 Hz」ラベル。
- 中央: 9 本の縦フェーダー(65 Hz〜16 kHz)。フェーダー上に現在ゲイン(0.0 dB)、下に周波数ラベル。目盛りは ±12 dB。
- 右: LPF ノブ + LED 式 ON/OFF ボタン。
- フェーダーはダブルクリックで 0.0 dB、Cmd/Ctrl ドラッグで 0.1 dB 刻み。

### 5.5 ディレイ / リバーブ

2 段ラック構成(参考画像 1)。

**上段: ECHO**
- 左: VU メーター(針アニメーション、30fps 以上)+ DELAY TIME / TEMPO ディスプレイ(黒背景、音価とBPM)。
- 中央: ACTIVE HEADS インジケーター(4 ドット、点灯で構成表示)+ MODE SELECTOR ノブ(4 ポジション、スナップ)+ SYNC 3 位置スイッチ(FREE / APP / TAP)+ TAP ボタン。
- 右: ノブ 2 段 ×4(TIME, FEEDBACK, TAPE-MOD, MIX / TAPE DRIVE, BASS, TREBLE, WIDTH)。BASS / TREBLE はアクセント色ノブで区別(参考画像のオレンジ)。
- 右端: POWER ボタン(点灯式)。
- SYNC=APP のとき TIME ノブは音価スナップ(1/1〜1/32、付点・三連含む)、FREE のとき ms 連続可変、TAP のとき TAP 入力値を表示。

**下段: REVERB**
- 左: ステータスディスプレイ(MODE / PRE-DELAY / DECAY / MIX / FILTER 帯域の数値をまとめて表示)。
- 中央: HALL / PLATE トグルスイッチ。
- 右: 縦フェーダー 5 本(PRE-DELAY, DECAY, LOW CUT, HIGH CUT, MIX)。フィルタ 2 本はアクセント色。
- 右端: POWER ボタン。

## 6. D. リンクウィジェット(下部中央)

参考画像 2〜4 の下部にある「🔗 + 3 ブロック」の UI。**アンプヘッドとキャビネットの組み合わせセレクタ**(確定仕様: spec §8.2 / §18-10)。

- 3 ブロック = **Clean / Crunch / Lead の 3 リグ**。各ブロック内の 2 つの丸「OO」= 左: アンプヘッド、右: キャビネット。選択中のアンプ / キャビは丸のハイライト(塗りつぶし)で表示。
- **🔗(リンク)ボタン ON**: アンプとキャビが**セットとして連動**。アンプを Crunch に切り替えると、キャビも自動的に Crunch 用へ切り替わる。
- **🔗 OFF(リンク解除)**: アンプとキャビを**自由に組み合わせ**られる(例: Clean アンプ × Lead キャビ)。アンプ側の丸とキャビ側の丸を個別にクリックして選択。
- リンクを OFF → ON に戻したときは、現在選択中のアンプに対応するキャビへスナップする(組み合わせの不整合を残さない)。
- 操作: 丸のクリックで直接選択。アンプタブ / キャビタブ内での切替操作とも**双方向に同期**する(どちらで替えてもウィジェットに反映)。
- 表示タブ: 全タブ常時表示(参考画像 1 で FX 画面にも表示されている)。
- プリセットには「リンク状態 + アンプ選択 + キャビ選択」を保存する。

## 7. 共通コンポーネント仕様

### 7.1 ノブ

| 項目 | 仕様 |
|---|---|
| サイズ | 大 72px(アンプ)/ 中 64px(ペダル・キャビ)/ 小 56px(グローバルバー) |
| 回転範囲 | -135°〜+135°(7 時〜5 時) |
| 操作 | 縦ドラッグ(上 = 増)。感度: 200px で全域 |
| 微調整 | Cmd/Ctrl + ドラッグで 1/10 感度 |
| リセット | ダブルクリックでデフォルト値 |
| 数値入力 | 右クリックメニュー「Enter Value...」+「MIDI Learn」「Reset」 |
| ホイール | マウスホイールで増減(1 ノッチ = 全域の 1%) |
| 値表示 | ノブ下に常時表示(dB / ms / st / 0.000 形式)。ドラッグ中はツールチップでも表示 |
| スナップ | TRANSPOSE(半音)、MODE SELECTOR(4 位置)等は離散スナップ + クリック触感(位置で吸着) |

### 7.2 フェーダー(EQ / REVERB)

- 縦ドラッグ、ダブルクリックでデフォルト、Cmd/Ctrl で微調整、ホイール対応。
- つまみ画像は上下 2 状態(通常 / ドラッグ中)。

### 7.3 スイッチ類

| 種類 | 用途 | 挙動 |
|---|---|---|
| トグル(横) | GATE, DOUBLER | クリックでON/OFF、ONは塗りつぶし |
| トグル(縦 2 位置) | INPUT MODE, HALL/PLATE | クリックまたはドラッグ |
| 3 位置スイッチ | SYNC (FREE/APP/TAP) | クリックした位置へ移動 |
| フットスイッチ | ペダル ON/OFF | クリックで踏み込みアニメ + LED |
| POWER ボタン | 各セクション | 点灯式ボタン。OFF 時はセクション全体を僅かにディム |
| PHASE ボタン | キャビ | Ø アイコン、反転中ハイライト |

### 7.4 ディスプレイ / メーター

- 7 セグ風数値ディスプレイ(黒背景 + アンバー/白文字): DELAY TIME, TEMPO, REVERB ステータス。
- VU メーター: 針の物理モデル(慣性 + ダンピング)で描画。UI アニメーションタイマー(SetDirty ループ)30〜60Hz。
- レベルメーター(グローバルバー両端): RMS + ピークホールド。

### 7.5 モーダル共通

- 背景を 40% 暗転、モーダルは角丸 12px の黒基調パネル。右上 ✕ で閉じる。Esc で閉じる。
- モーダル表示中も音声処理・下部バーは動作継続。

## 8. モーダル画面設計

### 8.1 チューナー(参考画像 3)

```
┌ Tuner ──────────────────────[🔇][✕]┐
│  -50      >      -1.7      <      +50 │  ← centスケール+左右矢印(高い/低いの誘導)
│  ────────────●────────────           │  ← インジケーター(±3cent以内で緑)
│  D#/Eb        E            F          │  ← 検出ノート(中央大、両隣小)
│  [Cents|Hz]  [ ]Live Tuner  [440.0▲▼] │
└──────────────────────────────────────┘
```

- インジケーター色: ±3 cent 以内 = 緑 / それ以外 = 白、フラット側・シャープ側の矢印(> <)が点灯して誘導。
- 🔇 ミュートボタン: 赤背景トグル。チューナーを閉じるとミュート自動解除。
- Live Tuner ON 時はミュートボタンをディム。
- 無入力時はノート表示を「—」。

### 8.2 メトロノーム(参考画像 4)

```
┌ Metronome ────────────────────[✕]┐
│ VOLUME   TIME SIGNATURE  SOUND        PAN │
│ (ノブ)    [4/4 ▼]        [Block..▼]🔊 (ノブ)│
├───────────────────────────────────┤
│ BPM                                       │
│ ▲▼ 120.0   [TAP]   RHYTHM [1/4 ▼]  [▶]    │
│      ● ○ ○ ○   ← 拍インジケーター           │
└───────────────────────────────────┘
```

- 拍インジケーター: 拍子に応じてドット数可変、1 拍目は大きく/アクセント色。
- 🔊 試聴: SOUND 変更時にワンショット再生。
- ▶ は下部バーの再生ボタンと状態同期。

### 8.3 SETTINGS

- タブ: Audio / General。
- Audio: ドライバ / 入力デバイス / 入力ch / 出力デバイス / 出力ch / サンプルレート / バッファ。下部に実測レイテンシ(in+out, ms)表示。プラグイン時は「ホストが管理」表示に差し替え。
- General: UI スケール(50〜200%)、起動時プリセット、アップデート確認。

### 8.4 MIDI マッピング

- パラメータ右クリック → MIDI Learn → 受信 CC を割当、トースト通知。
- MIDI ボタン → 一覧モーダル: [パラメータ名 | CC# | ch | 削除]。
- Learn 待機中は対象コントロールをアクセント色で点滅。

## 9. 状態表示ルール

| 状態 | 表示 |
|---|---|
| 未保存プリセット | プリセット名先頭に `*` |
| セクション OFF | セクション全体の彩度 -30% + POWER 消灯 |
| 操作不可(依存 OFF) | ディム(不透明度 40%)+ カーソル通常 |
| クリップ | メーター頂部 2s 赤点灯 |

| ドラッグ中 | ノブ/フェーダーに値ツールチップ |

## 10. 仮 UI → 本番 3D UI 差し替え方針

1. **スキン分離**: 独自 IControl を「挙動(当たり判定・パラメータ接続)」と「描画(スキン)」に分離して実装し、描画側を `FlatSkin`(仮)/ `SkeuoSkin`(本番)の 2 実装で差し替える。コントロール側は座標とパラメータ接続のみ持つ。
2. **レイアウトはデータ駆動**: 各画面の配置を JSON(コンポーネント名 / 座標 / サイズ / アセット ID)で定義し、本番画像に合わせて座標だけ再調整できるようにする。
3. **ノブ描画の 2 方式を両対応**:
   - フィルムストリップ(縦連結 PNG、61〜128 フレーム)
   - 単一画像 + 回転描画(ハイライトが固定光源の場合は不可)
   本番は 3D レンダのためフィルムストリップを基本とする。
4. **アセット命名規約**: `{screen}_{component}_{state}@2x.png`(例: `amp_lead_knob_gain@2x.png`)。Blender 側のレンダスクリプトと 1:1 対応させる。
5. **背景 1 枚 + 可動部オーバーレイ**: 各画面は背景 1 枚(機材全景)+ ノブ / スイッチ / LED / 針のみ個別アセット。
6. 仮 UI 期に**当たり判定(ヒットエリア)を本番想定サイズで確定**し、フェーズ 2 で操作感が変わらないようにする。

## 12. アクセシビリティ / ショートカット

- Cmd/Ctrl+1〜5: タブ切替 / T: チューナー / M: メトロノーム / Space(スタンドアロン): メトロノーム再生停止
- 全コントロールに Tab フォーカス + 矢印キー増減(iPlug2 のアクセシビリティ支援は限定的なため、フォーカス管理とキーボード操作は自前実装)
- 色のみに依存しない状態表示(ON/OFF はラベル・形状でも判別可能に)

## 13. iPlug2 (IGraphics) 実装マッピング(参考)
